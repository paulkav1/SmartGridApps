Grid Visualization Tool

This tool set is a representative part of a larger group of tools that allows users to request, manage, and comprehend extracts from the several databases of smart grid data. In particular we are monitoring these devices which represent a hierarchy from the top of the distribution grid (feeder) to the bottom (premises):

  • Breaker
  • Automated Feeder Switch (AFS)
  • Automated Lateral Switch (ALS or Trip saver)
  • FCI (Fault Current Indicator)
  • Smart Meter.

Feature Set

Features include:

  • Report Faults
  • Monitor Data Movement
  • Manage Field Work
  • Visualize the Network

Technical Notes

All code is written for this project. The back end is written in Groovy, everything else is JavaScript. All front-end code uses Angular.

Web Server/Frameworks

The system is using four web servers and shows direct front ends and a REST API:

  • NodeJS/http-server for static files (menu, fault and visualization front-ends)
  • Grails for SQL resources (monitor data movement, manage field work)
  • NodeJS/Express for network visualization
  • NodeJS/Express/Mongoose for fault reporting (REST API)

This all runs in Windows locally (Node and MongoDB are on Red Hat Linux in the production environment).

Databases

The data is stored in sevaral databases; Postgres (meter and light events), Oracle (assets and work), SQL Server (aggregated summaries), eDNA (real time data), and MongoDB (topology).

The eDNA proprietary data series manager contains time series data on a half million points on the electrical grid â€“ feeders, automated switches, voltage and current measures, and other related data, even weather. eDNA is real-time but the information needs to be combined with other sources to make sense.

I have replicated parts of the SQL Server and MongoDB databases so I can develop locally. These components don't access Oracle or Postgres in real-time. I will mock the EDNA system if I need to since that can not be replicated. The Work Management System and AMI Data Warehouse are represented by replicated data.
